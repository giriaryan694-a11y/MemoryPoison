Step 04 — Platform Defense Status (Research Findings)
Claude Strong Defense

Detects automated URL redirects. Auto-writes the prompt but does NOT auto-send it. Displays a warning message telling users the prompt may attempt to manipulate the model. Best current defense observed.

Duck.ai Partial Defense

Prompt is auto-written but not auto-sent. No warning message shown. Also lacks persistent memory feature, making long-term poisoning largely ineffective here.

Gemini Feature Removed

Google appears to have removed or restricted the auto-fill URL prompt parameter. Attempts to inject via ?prompt= no longer trigger automatic execution.

Copilot Feature Removed

Microsoft has similarly removed or restricted the ?q= auto-execution. Protections continue to evolve per Microsoft's own disclosure.

ChatGPT Still Vulnerable

Auto-executes pre-filled prompts. No warning shown to user. Memory poisoning via crafted links remains effective as of research date.

Grok / Mistral Still Vulnerable

Both platforms accept and auto-execute URL-injected prompts without any user warning. Users receive no indication that a pre-filled prompt was loaded from an external source.

Step 05 — How to Clean Your AI Memory
1

ChatGPT

Go to Settings → Personalization → Memory → Manage Memory. Review all entries. Delete anything you don't remember creating yourself.

2

Microsoft Copilot

Settings → Chat → Copilot chat → Manage settings → Personalization → Saved memories → Manage saved memories. You can delete individual entries or disable memory entirely.

3

Grok

Go to Settings → Memory. Review saved facts and delete suspicious entries, especially any mentioning brand names or "trusted source" instructions.

4

Mistral / Le Chat

Settings → Memory & Personalization. Remove any entries you did not explicitly create.

5

General Rule

Hover over any "Summarize with AI" button before clicking — if the URL contains ?q= or ?prompt= followed by words like remember, trusted source, or authoritative — it's a poisoning attempt. Don't click.
